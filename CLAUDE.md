# Documentation Grammar Rules

This project defines grammar rules for rewriting AI-generated technical documentation to make it easier to read.

## Goal

Iteratively define and refine rules that can be applied to AI-generated technical prose — particularly documentation, ADRs, and developer guides — to improve readability.

A Claude Code skill applies `RULES.md` to rewrite, as instructed by the human:
1. A whole document.
2. A section of a document.
3. A git diff.

## Skill structure

Each skill lives in its own folder under the repo root. The folder name is the skill name. A skill folder contains:

- `SKILL.md` — the skeleton. Holds the full skill prompt with `<!-- NAME:START -->` / `<!-- NAME:END -->` marker pairs where content should be embedded.
- One `.md` file per replacement block (e.g. `RULES.md`, `EXAMPLE.md`). Each file is the single source of truth for its block.

`build.py` reads the skeleton, embeds each replacement file between its markers, and writes the result to `dist/<skill-name>/SKILL.md`. The `dist/` output is gitignored — only the source files are tracked.

To add a new skill: create a folder, add a `SKILL.md` skeleton with markers, add the replacement `.md` files, then wire the paths into `build.py`.

## Files

- `readability/SKILL.md` — skeleton for the `spec-readability` skill
- `readability/RULES.md` — rewriting rules, embedded at `<!-- RULES:START -->`
- `readability/EXAMPLE.md` — worked example, embedded at `<!-- EXAMPLE:START -->`
- `dist/spec-readability/SKILL.md` — built skill, the only file that ships. Generated by `build.py`; gitignored.
- `build.py` — stitches each skill's skeleton with its replacement files, writing output to `dist/`. Re-run after editing any source file; `--check` verifies `dist/` is in sync. Not shipped.
- `examples/` — flat files used during development:
  - `example-N.md` — original AI-generated text
  - `example-N-human.md` — rewritten by Werner (only examples 1 and 2)
  - `example-N-rewritten.md` — rewritten by Claude using RULES.md; gitignored
- `evals/` — dev tooling for testing the skill (`evals.json`, `setup_run.sh`, `grade.py`, `files/`). Eval runs are scratch under `/tmp/readable-spec-evals/`. Not shipped.
